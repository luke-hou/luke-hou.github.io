3.# Line Bot MVP 專案作品集文案 (Modern AI Engineer Edition)
> **我是誰**: 一個喜歡寫程式解決身邊問題的工程師。
> **給誰看**: 每天被 LINE 訊息淹沒的一線人員。
> **核心理念**: 「科技不該是冷冰冰的代碼，而是幫大家省下時間、早點回家的溫柔力量。」這是我用 AI Vibe Coding 模式，一點一滴打造出來的團隊好幫手。

---

## 第一篇：把雜亂的 LINE 變成有序的時光機
**前言：**
「只要有用 LINE 的人就知道，在氾濫的賴群裡面掙扎有多痛苦。」
我不希望大家把時間浪費在「翻找歷史紀錄」上。所以我做了一個像時光機一樣的後台，把散落在各處的對話收好，讓訊息可以被管理，知識可以被累積，而不是隨著時間消逝在茫茫訊息海。

### 這個系統能幫團隊做什麼？
1.  **永久保存的雲端時光機 (Auto-Backup)**
    *   **所有格式通吃**：不管是截圖、錄影還是文字，系統都會自動備份到雲端 (Supabase)。
    *   **安心下載**：就算 LINE 顯示「檔案已過期」，你隨時回到後台，東西都在那裡。
2.  **看見真實的人 (Human-Centric Data)**
    *   **顯示真實姓名**：系統同步使用者的 LINE 顯示名稱，讓我們知道每則訊息背後都是一位真實的夥伴。
    *   **行內編輯 (Inline Edit)**：名字顯示不對？直接在網頁上點一下就能改，也能標記「排除統計」(例如機器人)，讓報表更乾淨。
3.  **找回專注力 (Focus Mode)**
    *   **一鍵過濾雜訊**：生活已經夠吵雜了。透過「Hide Ignore」開關，一鍵隱藏那些已處理完或不重要的訊息 (如「收到」、「謝謝」、跟負面情緒的語言)。
    *   **關鍵字高亮**：重要的字 (如 Bug、Urgent) 會自動變紅字粗體，掃視過去不會漏重點。
    *   **訊息搜尋體驗**：為了不浪費你的每一秒，搜尋框加入了「防抖 (Debounce)」設計。輸入關鍵字的瞬間，結果就已經準備好了。
4.  **決策者的戰情儀表板 (Visual Analytics)**
    *   **一眼看穿現況**：首頁儀表板直接顯示「今日訊息量」、「誰最忙」以及「待處理工單」。
    *   **趨勢圖表**：透過可收合的圖表區 (折線圖/圓餅圖/柱狀圖)，將抽象的「感覺很忙」轉化為真實的數據，這是你跟老闆爭取資源的底氣。

## 第二篇：架起 RD 與 PM 之間的友誼橋梁
**前言：**
「PM 辛苦整理需求，RD 辛苦修 Bug，但為什麼大家還是覺得資訊不同步？」
很多時候是因為工具斷層。我希望用這個系統把 LINE (溝通) 和 Jira (執行) 串起來，減少複製貼上的工作，讓大家合作更順暢。

### 如何讓協作變簡單？
1.  **一鍵轉工單**
    *   **附件自動搬家**：轉工單時，系統會自動把 LINE 裡的圖片/影片上傳到 Jira 附件區。RD 不用再問「截圖在哪裡？」，所有證據都在票裡。
    *   **防呆機制**：透過「Jira 智慧比對」，在開單前可以先查是不是有人開過了，避免重複工單造成的資源浪費。
2.  **殺手鐧：AI 自動生成工單**
    *   **多選一鍵生成**：勾選多則相關的對話與媒體檔案 (圖片/影片)，按一下按鈕，AI 自動讀懂上下文，瞬間生成一張結構完整的 Jira 工單，我們可以先預覽並編輯後再提交。
    *   **釋放腦力**：它不只是省下複製貼上的時間，更幫你省下「組織語言」的力氣。AI 幫你把破碎的資訊變成邏輯清晰的需求描述，你只需要做最後確認。
3.  **輕量級任務管理**
    *   **內部看板**：殺雞焉用牛刀，有些小事不用進 Jira，直接派工 Assignee 跟賦予狀態「New → WIP → Done」，我們直接在後台輕量化管理就好。
    *   **行內備註**：直接在表格裡寫 Note，就像在書頁邊緣寫筆記一樣自然。
4.  **購物車式暫存區**
    *   **拼湊完整需求**：客戶東講一句西講一句？先把相關訊息勾選起來放進「暫存區」，我們可以標記成一組任務方便歸檔跟追蹤處理，還可以透過 AI 幫你組合成一張邏輯完整的工單。

## 第三篇：你的 AI 守護者
**前言：**
「我們都有忙碌的時候，但系統問題與客戶情緒是隨時可能發生的。」
這就是為什麼我們要引入 AI。它不是要取代誰，而是像一個可靠的守護者，在我們無暇兼顧時幫忙看著。

### AI 實際能幫上什麼忙？
1.  **情緒溫度計 (Sentiment Monitor)**
    *   **情緒評分**：AI 會幫每個群組打分數 (0-100)，並用紅綠燈讓你一眼看出哪個群組需要關心。
    *   **情緒預警**：當 AI 感測到群組對話有「憤怒」或「失望」語氣，系統會主動亮紅燈。這能讓我們從「被動救火」轉為「主動關懷」。
2.  **活的知識庫 (Living Knowledge Base)**
    *   **自動萃取**：每一次精彩的技術排除，都可以用 AI 捕捉下來，自動生成「問題 + 解答」的 FAQ 知識庫。
    *   **人機協作**：我們提供完整的編輯器。你可以檢視 AI 生成的 FAQ 內容，修飾內容，讓資深前輩的智慧得以傳承給新人。
3.  **跨群組智慧比對 (AI Duplicate Check)**
    *   **過濾雜訊**：同樣的公告或 Bug 在五個群組都被轉貼？AI 幫你辨識出這是重複內容 (語意比對)，只保留一份最新的，閱讀一份就夠了。
    *   **視覺化報告**：系統會列出所有重複的訊息群組，讓你決定要「標記較舊」還是「保留最新」，大大節省閱讀時間。
4.  **主動式議題偵測 (Issue Discovery)**
    *   **比你先發現問題**：AI 不只是總結過去，還會幫你「預判未來」。它會主動掃描對話中潛在的 Bug 回報或功能建議。
    *   **防漏網之魚**：那些還沒被轉成工單的對話，AI 會自動識別並分類，列出「待辦事項清單」給你確認，確保沒有任何重要需求被遺忘。
5.  **溫暖的溝通助理**
    *   **一鍵催單 (AI Reminder)**：進度卡住了？按一下「催單」按鈕，AI 會讀取上下文，幫你寫出既專業又不失禮貌的詢問訊息，還會自動加上 Emoji 😊，我們只要一鍵貼上就好，不用像以前四處複製零散訊息再組合貼上。
    *   **自動化紀錄**：催單後系統會自動更新「最後催單時間」，下次你再看就知道「哦，這件事我兩小時前問過了」。
6.  **每日戰情摘要**
    *   早上進公司不用爬幾千層樓。AI 幫你讀完昨天的對話，濃縮成一份精簡的戰情報告，3 分鐘掌握全貌。

## 第四篇：因為兼職，所以更在意品質 (Automated Quality Assurance)
**前言：**
「身為一個部分工時的 SQA，我其實沒有時間把測項每一遍都手動測完。」
這聽起來很矛盾？不，正因為時間寶貴，所以更不能容忍重複的無效勞動。我堅持「能自動化的就不手動」，確保每一次的交付都是高品質的。

### 為什麼 MVP 也需要自動化測試？
1.  **速度與信心的平衡**
    *   **Test Runner**：這是我為這個專案寫的自動化測試執行器。從截圖中你可以看到，它能在 **13.86 秒內跑完 51 個測項**。
    *   **100% Pass Rate**：這代表每一次的 Code Change，我都確保它通過了所有的 API CRUD、權限認證、以及 Supabase 的整合測試。
    *   ![Test Runner Dashboard](assets/test_runner_dashboard.png)
2.  **可見的健康度**
    *   **Dashboard 整合**：我把測試結果直接整合進 Dashboard (參考上圖)。綠色的勾勾不只是好看，它是讓我跟團隊都能睡好覺的保證。
3.  **部分工時 QA 的堅持**
    *   我常常只有週末或下班能寫 Code。如果每次都要花 1小時手動回歸測試，我早就放棄了。
    *   這個自動化框架讓我敢在有限的時間內大膽重構 (Refactor)，因為我知道有 51 個忠實的守衛在幫我看著。

## 給夥伴的話
這個 MVP 的初衷，是希望透過技術和 AI 的力量，把繁瑣 Line 群管理的雜事用程式跟 AI 提供實際的幫助，讓我們都能把時間花在更有價值的地方。

這就是我用 AI Vibe Coding 想要傳達的價值：**務實、高效、以人為本。**

只要你敢想，AI 就能帶你飛。
