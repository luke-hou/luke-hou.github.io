PM x Gemini CLI (二)：Gemini CLI 基本指令與認證

發布日期：2025年10月

經過前篇文章的環境建置與專案設定，現在我們將正式開始與 Gemini CLI 互動！

文章大綱：

  • 步驟一：在專案資料夾中啟動 Gemini CLI
  • 步驟二：使用 Google Account 進行認證 (/auth)

  • 步驟四：/clear 指令的使用時機與說明

掌握這些基本操作，您就能開始利用 Gemini CLI 提升工作效率。

步驟一：在專案資料夾中啟動 Gemini CLI

開啟您的 Visual Studio Code，並確保您已經開啟了之前建立的專案資料夾 (例如 NHI-payment-guidelines)，以便與 Gemini CLI 一起協作。

接著開啟作業系統內建的終端機 (Terminal)，您需要先切換到您的專案資料夾。假設您的專案資料夾路徑為 C:\Users\YourUser\Documents\NHI-payment-guidelines (Windows) 或 /Users/YourUser/Documents/NHI-payment-guidelines (macOS)，請輸入以下指令：

Windows:

cd C:\Users\YourUser\Documents\NHI-payment-guidelines

macOS/Linux:

cd /Users/YourUser/Documents/NHI-payment-guidelines

切換到專案資料夾後，輸入 gemini 指令來啟動 Gemini CLI：

gemini

首次啟動時，Gemini CLI 會提示您進行認證。您也可以使用 --help 參數查看更多啟動選項：

gemini --help

這將顯示所有可用的啟動參數，例如設定模型、溫度等。

步驟二：使用 Google Account 進行認證 (/auth)

/auth 指令用於變更認證方式。當您首次啟動 Gemini CLI 時，它會提示您進行認證。您也可以手動輸入 /auth 指令來啟動認證流程：

/auth

認證流程詳解：

  • 當您執行 gemini 或 /auth 時，CLI 會生成一個唯一的設備代碼和一個驗證 URL。
  • 瀏覽器會自動打開該 URL，請授權 Gemini CLI 訪問您的 Google 帳戶。
  • 授權成功後，CLI 會自動獲取一個訪問令牌，用於後續與 Gemini API 的互動。
  • 如果遇到認證失敗或過期，您可以隨時重新執行 /auth 指令。

步驟三：/init 指令的使用時機與說明

/init 指令用於分析專案並建立一個客製化的 GEMINI.md 檔案。當您在一個新的專案資料夾中首次使用 Gemini CLI，或者您希望重置 Gemini CLI 的內部狀態時，可以使用這個指令。

執行 /init 指令後，Gemini CLI 會建立一些必要的設定檔和資料夾，以確保後續的操作能夠正常進行。這些設定檔通常包括：

  • .gemini-cli-config.json：存放 Gemini CLI 的配置，例如預設模型、溫度等。
  • .gemini-cli-history.json：存放您的對話歷史紀錄。
  • GEMINI.md：客製化的專案設定檔。
/init

使用時機：

  • 在新的專案資料夾中首次啟動 Gemini CLI。
  • 當您遇到 Gemini CLI 行為異常，希望重置其內部狀態時。
  • 當您希望清除之前的對話紀錄和上下文，重新開始一個新的對話時。
  • 當您需要自定義 Gemini CLI 的行為時，可以修改 .gemini-cli-config.json 或 GEMINI.md。

步驟四：/clear 指令的使用時機與說明

/clear 指令用於清除螢幕和對話歷史紀錄。這對於開始一個全新的對話，或者當您覺得之前的對話內容已經不再相關時非常有用。

執行 /clear 指令後，Gemini CLI 會忘記之前的所有對話內容，並回到初始狀態。

/clear

使用時機：

  • 開始一個全新的對話主題。
  • 當您希望 Gemini CLI 忽略之前的對話內容，避免其受到舊有上下文的影響時。
  • 在進行敏感資訊處理後，希望清除相關紀錄時 (請注意，這僅清除本地紀錄，不影響 Google 伺服器上的資料)。

與 /init 的區別：

  • /clear 僅清除當前會話的對話紀錄和上下文，不會影響配置檔。
  • /init 會重新初始化整個工作環境，包括配置檔和對話紀錄。

步驟五：其他常用指令

除了上述指令，Gemini CLI 還提供了一些其他實用的指令，可以幫助您更好地管理和使用它：

  • /help：顯示 Gemini CLI 的幫助資訊。
    • /help
  • /about：顯示版本資訊。
    • /about
  • /chat：管理對話歷史。您可以儲存、恢復、刪除或分享對話檢查點。
    • /chat save <tag>
    /chat resume <tag>
  • /docs：在瀏覽器中開啟 Gemini CLI 的完整文件。
    • /docs
  • /directory：管理工作區目錄。您可以新增或顯示工作區中的目錄。
    • /directory add <path>
    /directory show
  • /memory：管理記憶體內容。您可以顯示、新增、刷新或列出記憶體中的 GEMINI.md 檔案路徑。
    • /memory show
    /memory add <content>
    /memory list

Gemini 2.5 Pro 與 Gemini 2.5 Flash 的切換與應用

Gemini CLI 支援多種模型，其中 Gemini 2.5 Pro 和 Gemini 2.5 Flash 是兩個功能強大且各有側重的模型。了解它們的特性並適時切換，能讓您更高效地利用 Gemini CLI。

模型切換觸發機制

在實際使用中，您可能會基於以下情境考慮切換模型：

  • 免費額度考量： Gemini 2.5 Pro 通常會消耗較多的配額或有較高的使用成本。當您的 Pro 模型免費額度用盡，或希望節省成本時，可以切換到 Gemini 2.5 Flash。
  • 回應速度需求： Gemini 2.5 Flash 在設計上更注重速度和效率，對於需要快速獲得結果的任務，Flash 模型會是更好的選擇。
  • 任務複雜度： 對於複雜的分析、內容生成或需要深入理解上下文的任務，Pro 模型能提供更精確和全面的結果。而對於格式修正、簡單摘要等明確任務，Flash 模型足以勝任。
  • API 限制： 某些情況下，Pro 模型可能會遇到暫時性的 API 速率限制或服務不穩定，此時切換到 Flash 模型可以作為備用方案。

Gemini 2.5 Pro 適用情境

Gemini 2.5 Pro 是一個多功能、高性能的模型，擅長處理複雜的任務，提供深入的理解和高品質的內容生成。

  • 綜合參考資料與發想： 當您需要綜合大量文件、網頁內容或程式碼，並從中提取關鍵資訊、進行分析、生成新的想法時，Pro 模型表現卓越。例如，撰寫產品需求文件 (PRD) 時，它可以幫助您分析市場報告、用戶回饋，並提出創新的功能點。
  • 複雜問題解決： 對於需要多步驟推理、邏輯判斷或跨領域知識的問題，Pro 模型能提供更精確和全面的解決方案。
  • 創意內容生成： 撰寫部落格文章、行銷文案、劇本或任何需要高度創意和流暢表達的內容時，Pro 模型能生成更具吸引力和原創性的文本。
  • 程式碼生成與除錯： 生成複雜的程式碼片段、解釋程式碼邏輯、找出潛在的錯誤或提供優化建議。

Gemini 2.5 Flash 適用情境

Gemini 2.5 Flash 則是一個輕量級、高效率的模型，專為需要快速響應和處理簡單、明確任務而設計。

  • 文件格式修正與排版： 對於修正 Markdown 格式、調整程式碼縮排、統一文件風格等任務，Flash 模型可以快速完成，且成本較低。
  • 快速摘要與提取： 從較短的文本中快速提取關鍵資訊、生成簡短摘要或回答事實性問題。
  • 簡單的內容生成： 生成簡短的郵件、通知、社群媒體貼文等，這些內容對語氣和風格要求不高，但需要快速產出。
  • 即時互動與聊天： 在需要快速來回對話的場景中，Flash 模型能提供更流暢的互動體驗。
  • 初步篩選與分類： 對於大量數據進行初步的篩選、分類或標籤，例如將用戶回饋分為「Bug 報告」、「功能請求」等。

透過靈活運用這兩個模型，您可以根據任務需求，最大化 Gemini CLI 的效益。

PM 實用應用範例

重要提醒： 請勿將公司機密資料或個人隱私資料上傳至 Gemini CLI 進行處理。一旦資料上傳，即可能被用於模型訓練，且此過程不可逆。請務必確保您處理的資料符合公司政策與隱私法規。

使用 `@` 引用本地文件或資料夾：

作為產品經理，您可以利用 Gemini CLI 執行多種任務，提升工作效率：

Gemini CLI 支援使用 `@` 符號來引用本地的檔案或資料夾。這表示您可以將文件內容作為輸入，或者讓 Gemini CLI 處理整個資料夾中的文件。例如：

  • @filename.txt：引用單一檔案的內容。
  • @folder_name：引用整個資料夾中的所有文件內容。

這對於處理大量本地資料，例如多個使用者回饋文件、多份會議記錄等，非常方便。

  • 快速摘要會議記錄： 將會議記錄貼入 Gemini CLI，然後使用指令要求其生成摘要。您也可以直接引用本地文件：
    • 請摘要以下會議記錄的重點：
@meeting_notes.txt
  • 分析使用者回饋： 將大量使用者回饋整理成文本，讓 Gemini CLI 協助您分析情緒、提取關鍵問題或建議。您也可以直接引用本地文件：
    • 分析以下使用者回饋，並列出前三大痛點：
@user_feedback.md
  • 生成 PRD 初稿： 提供產品需求的基本資訊，讓 Gemini CLI 協助您生成 PRD 的初步框架或內容。您也可以直接引用本地文件：
    • 請為一個新的社交媒體功能生成 PRD 初稿，功能包括：發布動態、按讚、留言、分享。
@product_brief.txt
  • 腦力激盪新功能： 提供一個主題，讓 Gemini CLI 協助您發想新的產品功能或改進點。您也可以直接引用本地文件：
    • 針對提升用戶留存率，請提供 5 個新的產品功能點。
@brainstorm_ideas.md
