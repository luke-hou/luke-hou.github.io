Google Antigravity IDE 教學 (一)：環境建置與 Agentic Mode 初探

發布日期：2026年1月

歡迎來到 Agentic Coding 的新時代！

Google Antigravity IDE 不僅僅是一個編輯器，它是一個「Agent-First」的開發平台。在這裡，AI 不再只是提供建議的副駕駛 (Co-pilot)，而是能夠獨立思考、規劃並執行任務的代理人 (Agent)。本系列教學將帶您從零開始，掌握這個強大的工具。

為什麼 Antigravity 這麼強大？

傳統的 AI 輔助工具（如 Copilot）主要是「自動補全（Auto-complete）」，你寫一行，它猜下一行。但 Antigravity 的邏輯是「自動完成（Auto-complete Task）」。

  • 全域認知 (Global Context)： Agent 不是只看當前檔案，而是透過 Project Indexing 隨時掌握整個專案的架構、依賴關係與商業邏輯。
  • 多步推理 (Multi-step Reasoning)： 當你說「重構這個模組」時，Agent 會先分析影響範圍、規劃步驟、修改程式碼、最後執行測試，這是一個完整的思考迴路。
  • 工具使用 (Tool Use)： Agent 不只會寫 Code，還會跑 Terminal 指令、瀏覽網頁、甚至操作資料庫。
核心概念： 與傳統 IDE 不同，Antigravity 的核心在於對 AI Agent 下達「任務 (Task)」而非單純的編寫代碼。請準備好轉換您的開發思維！

步驟一：下載與安裝

目前 Google Antigravity IDE 處於預覽階段 (Preview)。請前往官方網站申請試用或下載安裝檔。

前往 Google Antigravity 官網

下載適用於 Windows 的安裝檔後，執行安裝程序。安裝過程與 VS Code 類似，會自動配置必要的環境路徑。

步驟二：登入與權限設定

首次啟動 Antigravity IDE 時，系統會要求您登入 Google 帳號以存取 Gemini 模型服務。

  1. 點擊歡迎畫面上的 "Sign in with Google"。
  2. 在彈出的瀏覽器視窗中完成驗證。
  3. 回到 IDE，您會看到右下角的連線狀態顯示為 Connected。
注意： Antigravity 深度整合了雲端推理能力，請確保您的網路連線穩定，並已啟用 Gemini Advanced 或相關企業方案以獲得最佳體驗。

步驟三：關鍵設定與環境調校

安裝完成並登入後，為了讓 Agent 發揮最大戰力，我們需要調整幾個關鍵設定。請點擊左下角的齒輪圖示 進入 Settings。

1. Project Indexing (專案索引)

這是 Antigravity 的「大腦」來源。請確保您的專案資料夾被正確索引。

  • 前往 Features > Indexing。
  • 將 "Deep Semantic Indexing" 開啟。這會讓 AI 不只做關鍵字搜尋，而是真正理解程式碼的語意。
  • 注意： 首次開啟大型專案時，索引可能需要幾分鐘，請耐心等待右下角的狀態列顯示 "Index Ready"。
2. Auto-Approval Levels (自動授權等級)

為了在「安全性」與「自動化」之間取得平衡，您可以設定 Agent 的權限等級：

  • Safe Mode (預設)： 任何修改檔案、執行終端機指令的動作，都需要人類點擊 "Approve"。適合新手。
  • Collaborative Mode： 允許 Agent 自動修改檔案與執行讀取類指令 (如 ls, cat)，但破壞性指令 (如 rm, deploy) 仍需審核。推薦設定！
  • Autonomous Mode： Agent 擁有完全權限。僅建議在測試環境或沙盒中使用。
3. .antigravityignore

就像 `.gitignore` 一樣，您可以在專案根目錄建立 `.antigravityignore` 檔案，告訴 Agent 哪些檔案或資料夾是「絕對禁區」（例如金鑰檔案、敏感個資）。

步驟四：認識雙介面 (Editor vs. Manager)

Antigravity 最大的特色在於它擁有兩種核心檢視模式，分別對應不同的工作層級：

1. Editor View (編輯器視角)

這是您熟悉的程式碼編輯介面，適合進行具體的程式碼修改與閱讀。在這邊，您可以像使用 VS Code 一樣操作檔案，但多了一個強大的 AI 側邊欄，隨時準備協助您修改當前檔案。

2. Manager Surface (管理者介面)

這就是 Antigravity 的靈魂所在。按下 Ctrl + M (或點擊頂部切換鈕) 進入此模式。

  • 任務看板 (Task Board)： 這裡顯示了 Agent 正在執行的任務狀態 (Planning, Executing, Verifying)。
  • Artifacts (產出物)： Agent 生成的計畫書、圖表、預覽連結都會匯總於此。
  • 人類介入 (Human in the loop)： 您可以在此審核 Agent 的實作計畫 (Implementation Plan)，批准或退回修改。

步驟五：發布您的第一個任務

讓我們來試試看 Agent 的能力。在 Manager Surface 中，點擊 "New Task"：

任務：請幫我建立一個簡單的 Python "Hello World" 網頁伺服器，並寫一份 README 說明如何執行。

您會看到 Agent 開始思考：

  1. Planning： 它會生成一份計畫，列出需要建立 `server.py` 和 `README.md`。
  2. Approval： 您點擊 "Approve" 同意計畫。
  3. Execution： Agent 自動建立檔案並寫入程式碼。
  4. Verification： Agent 嘗試在終端機執行伺服器，確認沒有報錯。

這就是 Agentic Workflow 的基本雛形！

內建能力：MCP 工具箱 (Built-in MCPs)

Antigravity 的強大之處在於它實作了 Model Context Protocol (MCP)。這讓 Agent 不再是封閉在聊天室裡的機器人，而是擁有「雙手」的工程師。

以下是您最常會用到的內建工具：

聯網搜尋 (Web Search)

當遇到未知的錯誤或是新出的 Library，Agent 會自動上網 Google，閱讀 StackOverflow 或官方文件，找出最佳解法。

檔案操作 (File Ops)

Agent 可以自由地瀏覽專案目錄 (list_dir)、讀取檔案內容 (view_file)，甚至進行跨檔案的重構 (replace_file_content)。

終端機控制 (Terminal)

安裝套件 (npm install)、執行測試 (pytest)、或是部署應用程式。Agent 能像您一樣操作 Shell。

Tips: 您不需要記住指令。只要說「幫我安裝 Flask」或「查一下這個 error」，Agent 就會自動選擇對應的工具。

核心觀念：模式與模型

在開始之前，您需要了解 IDE 介面上的兩個關鍵開關，這決定了您的 AI 副駕駛會如何工作。

1. 操作模式切換 (Mode Switching)

Antigravity 提供了兩種互動模式，您可以透過下方的狀態列或快捷鍵 Cmd/Ctrl + P 進行切換：

  • Chat Mode (聊天模式)：
    • 類似 ChatGPT 的問答體驗。
    • 適用時機：詢問語法、解釋程式碼、或進行簡單的一問一答。
  • Plan Mode (計畫模式 / Agentic Mode)：
    • 這是 Antigravity 的靈魂。AI 不會直接寫 Code，而是先產出一份 implementation_plan.md。
    • PM 必用：這就像是您請工程師寫 Code 前，要求他先提「規劃書」。您可以在這階段審核邏輯，同意後 AI 才會開始執行。
2. 模型選擇 (Model Selection)

您可以根據任務的難易度，選擇不同的「大腦」：

模型 (Model) 特性 PM 應用場景
Gemini 3 Pro 高智商 慢思維 撰寫 PRD、分析複雜客訴、規劃大型功能架構。它能處理大量的 Context (上下文)，適合需要深度推理的任務。
Gemini 3 Flash 速度快 低延遲 翻譯文件、簡單的 Bug 修復、快速查詢指令。當您需要「秒回」的體驗時使用。
