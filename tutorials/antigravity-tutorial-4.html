Google Antigravity IDE 教學 (四)：建立 Agentic Workflows (標準化流程)

發布日期：2026年1月

從「會用工具」到「遵守流程」

在前面的教學中，我們學會了 SKILL（技能），這就像是教會 Agent 使用「鐵鎚」。但如果我們現在要蓋房子，光有鐵鎚是不夠的，我們需要的是藍圖與施工順序。

這就是 Workflows 的用途：定義一套標準作業程序 (SOP)，強迫 Agent 按照我們規定的順序，一步步完成複雜任務。

概念升級：領域專家 vs. 專案經理

在 Antigravity 的生態系中，我們不應該只把 Skill 當作工具，而應該視為「專家」。

特性 Agent Skill (技能) Agent Workflow (流程)
角色定位 領域專家 (The Specialist) 專案經理 (The Manager/SOP)
核心能力 擁有特定領域的深度知識與判斷力 (Intelligent) 擁有掌控全域進度與順序的權力 (Orchestration)
比喻 「資深資安工程師」(會看 Code 找漏洞) 「年度資安稽核流程」(規定何時掃描、何時修補)

Workflow 的真諦： 它不是用來限制 Agent，而是用來「指揮」多位專家協同合作。我們將在下一章深入探討這兩者如何產生 Synergy (綜效)。

步驟一：建立 Workflow 檔案

Workflows 存放於 .agent/workflows/ 資料夾中。讓我們來建立一個 PM 每天都要做的流程：用戶回饋分析 (Feedback Analysis)。

mkdir -p .agent/workflows/analyze_feedback.md

這個 Workflow 將會幫助你從茫茫的客服回饋中，萃取出最有價值的洞察：

---
description: 自動分析用戶回饋 CSV 並產出洞察報告
---
1. 讀取回饋資料 (Read Feedback)
   - 讀取 `raw_data/feedback_export.csv`。
   
2. 分類與貼標 (Categorize)
   - 逐條分析回饋內容。
   - 標記為 [Bug], [Feature Request], 或 [UX Issue]。
   - 判斷情緒分數 (Sentiment Score)。

3. 總結洞察 (Summarize Insights)
   - 統計 Top 3 最頻繁出現的痛點。
   - 挑選 5 條最具代表性的使用者原音 (Quotes)。
   - 輸出報告至 `reports/weekly_insight.md`。

實戰範例：產品規格視覺化 (PRD to Mockup)

PM 最常遇到的痛點是「文字規格過於抽象」。讓我們建立一個 Workflow，自動把 PRD.md 文字規格轉成可互動的網頁 Mockup，並整理出驗收重點。

mkdir -p .agent/workflows/visualize_prd.md
---
description: 將 PRD 文件轉換為 HTML Mockup 與摘要
---
1. 分析 PRD 文件 (Analyze PRD)
   - 讀取 `docs/PRD.md`。
   - 擷取「核心功能」與「UI 需求」。

2. 生成視覺化原型 (Generate Mockup)
   - 根據 UI 需求，編寫一個單頁的 HTML/Bootstrap 原型。
   - 儲存為 `mockups/v1_preview.html`。

3. 產出重點摘要 (Summarize)
   - 條列出 PRD 的驗收標準 (Acceptance Criteria)。
   - 將摘要與 Mockup 連結寫入 `PRD_Summary.md`。

步驟二：Turbo Mode (極速模式)

如果某些步驟你非常有信心，不希望 Agent 每次都停下來問你「可以執行嗎？」，你可以使用 // turbo 註解。

1. 執行單元測試
// turbo
2. 建置 Docker 映像檔
   - 這是安全的操作，不需確認。
Turbo Tip: 加上 // turbo 後，Agent 只要確定指令安全 (SafeToAutoRun)，就會自動連續執行，直到遇到需要人類審核的關鍵步驟為止。

步驟三：執行 Workflow

在 Manager Surface 或對話框中，您可以直接指定要跑哪個流程：

「Agent, 請幫我執行 deploy_staging 流程。」

Agent 就會讀取該 Markdown 檔案，將每一個編號項目視為一個子任務 (Sub-task)，依序執行並回報進度。

系列結語

從環境設定、技能擴充，到今天的標準化流程，您已經完整掌握了 Agentic Coding 的核心三要素。現在，您可以讓 AI 不僅僅是寫程式，更是您團隊中遵守紀律、執行力強大的最佳隊友。
