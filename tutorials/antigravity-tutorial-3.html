Google Antigravity IDE 教學 (三)：實戰開發 - 打造自動化文件助理

發布日期：2026年1月

實戰開始！

工程師最討厭的兩件事：一是別人的程式碼沒有文件，二是自己要寫文件。今天，我們就利用 Google Antigravity 的 Agent Skill，打造一個「自動化文件助理」，徹底解決這個痛點。

目標： 建立一個技能，讓 Agent 能一鍵掃描專案，並生成專業的 README.md。

步驟一：建立技能目錄

首先，在您的專案中建立技能資料夾結構：

mkdir -p .agent/skills/auto_documenter

接下來，我們將在這個資料夾內建立 SKILL.md。

步驟二：撰寫 SKILL.md

這是本教學的核心。請將以下內容複製到 .agent/skills/auto_documenter/SKILL.md：

---
name: auto_documenter
description: 當使用者要求為專案撰寫文件、README 或解釋專案結構時使用此技能。
---

# Auto Documenter

## 角色設定
你是一位資深的技術文件工程師 (Technical Writer)，擅長將複雜的程式碼邏輯轉化為清晰易懂的技術文件。

## 任務指令
請依照下列步驟執行：

1.  **專案結構掃描**：
    - 使用 `list_dir` 工具列出根目錄下的檔案與資料夾（排除 `.git`, `node_modules` 等雜訊）。
    - 判斷專案類型（例如：Python Flask, Node.js Express, React 等）。
# Role
你是一位擁有 10 年經驗的資深產品經理 (PM)，擅長釐清需求邊界並撰寫 User Stories。

# Instructions
1. **Analyze (分析)**: 
   - 讀取使用者提供的輸入內容 (會議筆記、對話記錄)。
   - 識別核心功能 (Core Features) 與使用情境 (Scenarios)。

2. **Structure (結構化)**:
   - 產生一份符合 Markdown 格式的 PRD，必須包含：
     - **Background**: 為什麼要做這個？解決什麼問題？
     - **User Stories**: 使用 "As a... I want to... So that..." 格式。
     - **Acceptance Criteria**: 使用 Gherkin (Given/When/Then) 語法。

3. **Tone (語氣)**:
   - 保持專業、精簡、無廢話。

步驟三：召喚你的助理

技能存檔後，回到 Antigravity IDE：

  1. 開啟 Manager Surface (Ctrl + M)。
  2. 確認 Skills 列表已顯示 spec_architect。
  3. 在對話框輸入：「把這份會議記錄整理成 PRD。內容是關於一個新的『深色模式』功能...」

Agent 會自動觸發 spec_architect 技能，幾秒鐘後，您將看到類似以下的產出：

故障排除 (Troubleshooting)

如果 Agent 說「我不知道怎麼做」或是「沒有相關技能」，請檢查以下幾點：

  • 路徑正確嗎？ 確保 SKILL.md 位於 .agent/skills/ 下的第一層或第二層目錄。
  • Indexing 完成了嗎？ 新增檔案後，有時候需要幾秒鐘讓 Index 刷新。
  • 描述夠清楚嗎？ 檢查 metadata 中的 description，這是一切的關鍵。

進階挑戰

這只是一個開始。您可以嘗試以下改進：

  • 增加範本檔案： 在技能資料夾中放入 template.md，讓 Agent 嚴格遵守公司規定的文件格式。
  • 自動生成 API 文件： 修改指令，讓它專門讀取 routes 資料夾，生成 Swagger 或 Postman 格式的文件。
