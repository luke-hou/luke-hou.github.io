建立時間		@2025年8月14日 下午9:57
標籤

複製並改造舊工作流

打開先前做過的 Supabase Embeddings 工作流

（如果沒參與到的，請花幾分鐘閱讀並建立工作流：)

點選右上角的選單後選 Duplicate

我們從原本的克隆一個新的出來修改

按下去之後，會彈出一個小視窗

這邊我們先輸入 “Local_Qdrant_Embeddings”

按下 Duplicate 後會開出一個新的瀏覽器分頁

接下來我們都在這個新的分頁工作，原本的那個可以關了。

緊接著我們刪除這兩個 Supabase 的節點

在右邊點加號

新增一個 Qdrant 半官方節點(非n8n官方製作但是經過 Qdrant 官方背書的)

這邊如果沒安裝過的，要先點選 Install

已安裝過的就會跟我截圖一樣

安裝好後往下拉 找到 Delete Points

接著一樣依照慣例先設定 Credential

API Key 留空

REST URL : http://127.0.0.1:6333

點右上的 Save

正常會出現綠色的成功提示，失敗的話我也沒辦法。

依照下圖，依次設定好節點的參數內容。

Filter 切換到 Expression 後貼上這串，其他欄位則保持不變。

{
  "must": [
    {
      "key": "metadata.file_name",
      "match": {
        "value": "{{ $('On form submission').item.json.data.filename }}"
      }
    }
  ]
}

關閉刪除節點

再次於 Extract from File 點加號，搜尋 qdrant，點選 Qdrant Vector Store

繼續選 Add documents to vector store

點開節點，依照圖片進行設定。

然後 ESC 關閉節點

接著把紅框框的 Embedding 拉到 Google 那顆球，然後連接起來。

然後點開設定為 models/gemini-embedding-001

接著把紅框框的右腳連接到 Default Data Loader

這時候紅框警示都消失了，我們執行看看吧。

一樣選我們的老朋友 最新版藥品給付規定內容(整份帶走)-114.02.24更新

正常來說這樣就跑完了

回到 Qdrant Dashboard 查看資料

http://127.0.0.1:6333/dashboard#/collections

現在可以看到 Points 的數字跑出來了

再跑一次工作流，丟進去同一份檔案，確認 Points 數量還是一樣 688

代表 Delete 節點有生效

接著直接點選數字就可以看到首筆嵌入的資料內容了

繼續把所有珍藏的健保相關資料都丟進去吧！

這次能限制你的只剩硬碟空間了
