AI n8n 教學 (一)：基礎概念與安裝

發布日期：2023年9月10日

歡迎來到 AI n8n 教學系列的第一部分！本篇教學將帶您了解 n8n 的核心概念、主要功能以及如何在您的環境中安裝 n8n。

1. 什麼是 n8n？

n8n 是一個強大且可擴展的開源工作流程自動化工具。它允許您將不同的應用程式和服務連接起來，自動化重複性任務，而無需編寫程式碼。

  • **視覺化工作流程**：透過拖放介面輕鬆建立複雜的自動化流程。
  • **豐富的整合**：支援數百種應用程式和服務的整合。
  • **自託管選項**：您可以將 n8n 部署在自己的伺服器上，完全掌控您的資料。
  • **可擴展性**：支援自定義節點，滿足特定需求。

2. n8n 的核心概念

  • **節點 (Nodes)**：每個節點代表一個應用程式、服務或一個操作（例如 HTTP 請求、資料轉換）。
  • **工作流程 (Workflows)**：由多個節點連接而成的自動化流程，定義了資料如何流動和操作。
  • **觸發器 (Triggers)**：工作流程的起點，可以是定時任務、Webhook 接收、或特定事件發生。

3. n8n 安裝指南

n8n 提供了多種安裝方式，最常見的是使用 Docker 或 npm。

使用 Docker 安裝 (推薦)

Docker 是安裝 n8n 最簡單且推薦的方式。確保您的系統已安裝 Docker 和 Docker Compose。


# 創建一個目錄用於存放 n8n 資料
mkdir ~/.n8n
cd ~/.n8n

# 創建 docker-compose.yml 文件
# 將以下內容複製到 docker-compose.yml
version: '3.8'

services:
  n8n:
    image: n8nio/n8n
    restart: always
    ports:
      - "5678:5678"
    volumes:
      - ./n8n_data:/home/node/.n8n
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=user
      - N8N_BASIC_AUTH_PASSWORD=password
      - N8N_EDITOR_BASE_URL=http://localhost:5678/
      - WEBHOOK_URL=http://localhost:5678/

保存文件後，在同一目錄下執行：

docker-compose up -d

n8n 將會在 `http://localhost:5678` 啟動。

使用 npm 安裝

如果您有 Node.js 環境，也可以使用 npm 安裝 n8n：

npm install -g n8n

然後啟動 n8n：

n8n start
